## 0. 현재 상태 (전수조사, 2026-07-02)

**NPC 맵 이동**
- 로직상 NPC는 `turtle`/`frog` 2종뿐(`backend/sim/game_run.py:31-34`, `_NPC_SCHEDS`/`_NPC_STIM_TRAP`) — §9.5.3의 7종(패닉셀 개미·FOMO 단타러·가치투자자 등) 중 어느 것도 구현 안 됨.
- 맵(`backend/static_demo/map.html`)엔 스프라이트가 클론(`gamerun_clone`, `Isabella_Rodriguez` 리스킨) **1개뿐**. `turtle`/`frog`는 시각 자산조차 없음(전용 스프라이트 없음).
- 클론은 실제로 걷는다(`playWalk()` → `/control/game/day/walk` → 타일별 이동, 진짜 동작 확인).
- (참고) 예전 reverie 구엔진은 6에이전트가 맵을 다녔으나 이번 세션 초반에 승인받아 전부 삭제함 — 지금 새로 만드는 건 그 부활이 아니라 새 GameRun 구조 위에 새로 얹는 것.

**게시판/FGI**
- `backend/sim/social.py`의 `fgi_post()`는 톤(calm/clarify/fear_join)+roll → `crowd_mood` 실수 하나를 ±2.0 흔드는 게 전부. **텍스트 콘텐츠 전혀 없음.**
- `PhoneModal.tsx`의 "💬 피드" 탭 = 톤 3개 드롭다운 + 버튼 + 결과 숫자 표시. 게시글·댓글·피드 UI 없음.
- FGI는 언제든 선택적으로 여는 사이드 액션 — 하루 진행을 막지 않음.
- §9.5.4가 스펙한 `SNS_PERSONA_POOL`(관중 8종, 맵 미등장 by design) — `sns.py`/`personas.py` 등 코드베이스에 아예 존재 안 함.

## 2. 결정 로그

**D1 — 게시판 발생 조건: 이벤트 있는 날만 (평온한 날은 스킵)**
- 근거: 매일 발생시키면 이미 60초+ 걸리는 하루 진행(T-219)에 30일 내내 또 다른 블로킹 모달이 얹혀 페이싱이 무거워짐. 시그니처 드라마 전용은 너무 희귀해 "클론도 여기 참여" 재미가 옅어짐.

**D2 — 트리거 정의: ~~위기 발동 OR~~ 고강도 뉴스 OR 시장 급변동 (D12로 개정)**
- ~~위기 발동일(`snap.trap is not None`) 이거나,~~ 그날 뽑힌 뉴스 중 intensity=high/extreme인 게 있거나, **주력 종목이 전일 대비 ±8%(`BOARD_MARKET_MOVE_PCT`, [TUNE]) 이상 움직인 날**(급락→fear, 급등→fomo) 게시판이 열림. 함정(위기)은 트리거에서 제외 — D12 참고.
- 파생 작업: 현재 `news_pool.seed.json`의 일반 뉴스 43개는 `intensity` 필드가 없음(시그니처 드라마만 있음) — 일부 뉴스에 high/extreme 태깅 추가 필요.

**D3 — 배치 순서: 게시판 먼저 → 위기카드, 단 뉴스선택 직후가 아니라 하루진행 낮 단계에서**
- 뉴스 3지선다에서 고르는 그 순간(📰 버튼)이 아니라, "하루 진행▶" 눌러 아침→낮으로 넘어가는 도중에 발동. 기존 `crisis_check`가 이미 이 타이밍(☀️ 한낮 stage)에 계산됨 — 같은 호출 결과를 게시판 트리거 판정과 위기카드 표시 양쪽에 재사용(부작용 없는 `preview_crisis`라 이중 호출 안전).
- 순서: 🌅 아침 → [게시판 트리거 판정] → (열리면) 📱 게시판 교류 모달, 닫을 때까지 진행 정지 → ☀️ 한낮 → (위기면) 🧭 위기카드 → 🌇 저녁 → 정산.

**D4 — NPC 맵 이동 + 게시판을 이번에 같이 간다**
- §9.5.3 매매 에이전트(맵 등장·이동) + §9.5.4 관중(게시판 전용)을 같은 세션에서 함께 구현. 스코프가 커짐(신규 스프라이트/동선/1:1 만남 판정까지) — 아래 결정으로 초기 페르소나 수를 조절해 범위를 관리.

**D5 — 페르소나 수: 15종 전부 (§9.5 그대로, 축소 없음)**
- 매매 에이전트 7종 + 관중 8종 + 클론. 스코프 큼 — 구현 시 여러 하위 티켓으로 쪼갤 것(스프라이트/동선 자산화, 페르소나 데이터, LLM 프롬프트, UI 순).

**D6 — 게시판 모달 안에서는 관찰만, 개입은 기존 핸드폰 피드 탭에 그대로 유지**
- 새 게시판 이벤트 모달 = 순수 관찰(오늘 군중이 뭘 겪었는지 거울처럼 봄). 익명 톤 개입(§9.2.3, "휩쓸리지마/같이 무서워하기/차분히 짚기")은 지금처럼 `PhoneModal`의 "💬 피드" 탭에서 언제든 별도로 가능 — 두 기능이 공존.

**D12 — 게시판 주제 재정의: "마을의 공개 이벤트에 대한 감정 수다" (사용자 재정의, 2026-07-02 L3 리뷰 후)**
- 배경: /review 적대적 리뷰가 "아침 게시판의 클론 게시글이 preview_crisis 기반 합리화 문구라 **오늘 터질 함정을 누설**(§12.4 비공개 위반) + 버틴 날에도 휘둘린 것처럼 박제"를 발견 → 사용자에게 물었더니 근본 정의가 어긋나 있었음: **게시판은 클론의 함정이 아니라 그날의 공개 이벤트(뉴스·시장)에 대한 에이전트들의 감정적 대화다.**
- 개정 내용: ①트리거·컨텍스트에서 함정 완전 제거(D2 개정 — 시장 급변동 or 고강도 뉴스만, `board_trigger` 시그니처에 trap 자체가 없음을 테스트로 봉인) ②클론 게시글은 합리화 언어 대신 **이벤트에 대한 감정 반응**(D9 개정 — 관련 스탯<50이면 동요/≥50이면 침착 문구, `board_pool.seed.json`의 `clone` 섹션. 수치는 규칙·문구는 표현 §12.0). 클론은 매 게시판 참여(D9 원안 유지).
- 같은 리뷰에서 고친 것(D11): crowd_mood 여파를 관찰 GET이 아니라 advance_day 밤 정산에서 1회 적용 — crisis_check 레이스·동시호출 이중적용·네트워크 유실로 인한 결정론 붕괴 차단.

**D10 — 게시판 발생률: 일 ~67%(뉴스발)+위기일 용인 (사용자 결정, 2026-07-02)**
- T-225 검증 중 실측: 톤별 1개씩 3개를 뽑는 구조라 "high/extreme 포함" 확률이 아이템 비율 30%가 아니라 **일일 67%**(10,000회 시뮬레이션, crc32 시드 편향 없음 확인). D1 페이싱 우려로 사용자에게 포크 제시 → **"자주 열려도 된다"로 용인, 트리거 규칙·뉴스 풀 무변경.** 플레이테스트(T-SVC7)에서 무겁게 느껴지면 뉴스 풀 high 축소(fear2/opt1/unc1 → ~25%)로 재조정 가능하도록 근거 수치를 여기 남긴다.

### 3.1 NPC 맵 이동 — 매일 상시(이벤트 트리거와 무관), §9.2.1b 그대로

- **자산 재사용**: 신규 스프라이트 제작 불필요 — `environment/frontend_server/static_dirs/assets/characters/*.png`에 클론이 이미 쓰는 `Isabella_Rodriguez` 외 16종이 남아있음(Abigail_Chen, Adam_Smith, Klaus_Mueller 등). 매매 에이전트 7종에 이 중 7개를 배정해 리스킨(클론과 같은 방식).
- **스케줄**: `turtle`/`frog`의 `_NPC_SCHEDS` 패턴을 7종으로 확장 — 30일 회차 내내 고정, 매일 다른 장소(§9.2.1 "NPC 전원의 30일 일과는 고정").
- **1:1 만남**: `overlap_meetings()`(`day_loop.py`)가 이미 "클론 일과 vs 전 NPC 일과 겹침" 계산 — 2종→7종으로 그대로 확장. §9.2.1b "겹치면 클론이 취약점 강한 쪽으로 끌린다" 선택 로직 신규 추가(`clone_picks_npc` 확장, 수치 기반이라 §11.4/§8.5와 같은 판정 재사용).
- **시각화**: `map.html`의 `sprites{}` 딕셔너리를 클론 전용 1개 → N개로 확장. 매일 홈 위치 로드 시(`/control/game/day/home`) 그날 각 NPC의 목적지도 함께 내려받아 `moveSprite()`로 걷기 재생(클론과 동일 메커니즘, 그저 대상이 늘어남).
- **가격 영향 없음**: 순수 연출 — §6.1 "블라인드 실데이터" 원칙 유지, 매매 에이전트가 실제로 주문을 넣어 가격을 바꾸진 않음(§9.5.3 표의 "가격에 영향"은 감정 전염을 통한 간접 영향이지 직접 주문 체결이 아님 — 기존 §6.4 "뉴스→가격 간접 경로"와 같은 절제).

### 3.2 게시판(SNS형 FGI) — 이벤트 있는 날만, 하루진행 블로킹

**트리거 판정 (백엔드, `GameRun`에 새 메서드 또는 `preview_crisis` 확장)**
```
board_trigger = (crisis_check.trap is not None) or any(n.intensity in ("high","extreme") for n in today_news_drawn)
```

**콘텐츠 생성 (§9.4 "수준2 하이브리드" 그대로 재사용 — decisions=rule, expression=LLM opt-in)**
1. 오늘의 트리거(함정 종류 또는 뉴스 톤)에 맞는 관중 3~5명을 §9.5.2 감정축 적합도로 결정론 선정(예: F1 → 공포팔이·코린이뉴비·주식방 악플러 우선순위↑).
2. 각 선정 관중에 템플릿 문구 1개 배정(오프라인 기본 — LLM 키 없어도 항상 동작).
3. 선정 관중 중 1~2명에 "댓글" 배정(다른 관중이 답글 — 상호작용처럼 보이게, 결정론 규칙).
4. **클론도 게시**: 그날 `trap`이 있었으면 `clone_spec.rationalization[trap]`(이미 있는 합리화 언어, §12.0/§8.6.3 narrate()가 쓰는 것과 동일 소스) 재사용해 클론 명의 게시글 1개 추가. 평온한 날은 트리거 자체가 없으므로 클론 게시 없음.
5. `use_llm=True`면 2·4에서 만든 각 텍스트를 `_llm_rewrite`와 같은 패턴(개별 문장 다듬기, 실패 시 템플릿 폴백)으로 대체. **수치 판정(피드 구성·감정 델타)은 LLM과 무관 — §8.6.3 원칙 유지.**
6. `crowd_mood`는 생성된 피드의 지배적 톤에 따라 자동으로 소폭 이동(플레이어 조작 아님 — 관찰 이벤트라 §9.2.3의 수동 개입과는 다른 경로). 기존 `fgi_post()`는 그대로 두고 별도 함수로 구현(수동 개입 경로 안 건드림).

**데이터 계약**
```json
{
  "trigger": "F1" | "news:opt_inflow" | ...,
  "posts": [
    {"author": "공포팔이", "author_kind": "sns", "text": "...",
     "comments": [{"author": "코린이 뉴비", "text": "..."}]},
    {"author": "클론(run1)", "author_kind": "clone", "text": "...", "comments": []}
  ],
  "crowd_mood_delta": -3.5
}
```

**API**: `GET /control/game/day/board?game_id=...` — `crisis_check`처럼 부작용 없는 프리뷰가 아니라, 호출 시점에 그날의 피드를 확정 생성(같은 날 재호출해도 같은 시드로 같은 결과 — 결정론 유지). 프론트가 "게시판에서 교류중…" 모달을 띄운 채 이 호출을 기다렸다가 완료되면 피드를 보여줌.

**하루진행 흐름 삽입** (`frontend/app/play/page.tsx`의 `handleAdvance`)
```
🌅 아침 → crisis_check 1회 호출(위기+게시판 판정 겸용)
  → board_trigger면: setBoardOpen(true) + await api.gameBoard(...) → 모달에 피드 표시 → 사용자가 닫을 때까지 대기(await, Promise resolve on close)
☀️ 한낮 → (crisis_check.trap 있으면) 기존 CrisisEventModal
🌇 저녁 → 기존 finishAdvance
```
