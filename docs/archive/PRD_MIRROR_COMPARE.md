## 1. 스코프

§13.6 회차 비교 화면 1개 + 진입 경로 배선. 하드모드는 티켓만 적립(`[draft]`).

## 2. 결정 로그

**C1 — 비교 대상: 자동(직전 완주 회차 vs 이번) 기본 + 드롭다운으로 임의 회차 선택**
- §13.1 후회 서사는 "직전을 뒤집었나"가 기본 질문. 수동 선택은 컬렉션이 쌓인 뒤의 부가 기능이라 드롭다운 하나로 충분.

**C2 — 비교할 날: 30일 나열이 아니라 "분기일 자동 추출"**
- §13.3 "결과 아는 날은 빨리감기" — 두 회차의 스냅샷이 갈린 날(`swayed`/`fund_flow`/`companion` 중 하나라도 다름)만 골라 보여준다. 백엔드에 분기일 추출 헬퍼 신설(`runs.py`, 스냅샷 비교 순수함수 — 계약 무변경).
- 분기일 0개(완전 동일 플레이)면 "두 회차가 같은 선택을 했습니다" 빈 상태 표시.

**C3 — 진입 경로: `/report` 완주 직후 + `/history` 카드에서 — 두 곳 다**
- 카타르시스(§13.2 장치3)는 완주 직후가 골든타임 → `/report` 하단에 "🪞 지난 회차와 비교"(완주 2회차부터 노출). `/history`는 회고용 재진입 경로.

**C4 — 카타르시스 연출은 화면 안 인사이트 라인으로(별도 연출 티켓 없음)**
- §13.6 와이어프레임의 `💡 같은 급락, 다른 선택, 다른 결과` 라인을 분기일마다 규칙 기반 생성(예: 전회차 swayed & 이번 버팀 & 결과 개선 → "뒤집기" 문구). LLM 불요(표현 계층 최소, §12.0 정합).

## 3. 유저플로우 · 와이어프레임

**진입 흐름**

```
Day30 완주 → /report (결과 카드)
              └ [🪞 지난 회차와 비교]  ← 완주 2회차부터 노출 (C3)
                    ↓
/history (카드 컬렉션)                /compare?a=<직전>&b=<이번>
  └ 카드마다 [비교] 버튼 ─────────────↗
```

**회차 비교 화면 (`/compare`, 신규)**

```
┌─ 🪞 회차 비교 — 이번(회차3) vs [회차2 ▾] ──────────────┐
│  분기일 3개를 찾았습니다: [Day 6] [Day 14] [Day 22]      │  ← C2 자동 추출, 탭
│ ┌─ Day 14 ──────────────────────────────────────┐ │
│ │              회차2              회차3(이번)       │ │
│ │  행동:       😱 휩쓸림(F1)      🪨 버팀           │ │
│ │  교류:       패닉셀 개미와       가치투자자와       │ │
│ │  자금흐름:   현금 도피          유지               │ │
│ │  결과:       실현손익 -22       총자산 +8          │ │
│ │ ─────────────────────────────────────────────  │ │
│ │ 💡 같은 날, 다른 선택 — 이번엔 버텼고, 반등을 봤다.  │ │  ← C4 규칙 기반
│ └────────────────────────────────────────────────┘ │
│  (분기일 0개면: "두 회차가 같은 선택을 했습니다.")        │
│                              [컬렉션으로] [새 회차 ▶]  │
└──────────────────────────────────────────────────────┘
```

- 데이터는 기존 `GET /control/game/compare`(day별 a/b 뷰) 그대로 소비. 분기일 목록만 신규(§2 C2).
- 행동/교류 라벨은 스냅샷 원값(`swayed`·`companion`·`fund_flow`)의 한국어 라벨링 — `DayResultToast`의 `FUND_FLOW_LABEL` 재사용.
