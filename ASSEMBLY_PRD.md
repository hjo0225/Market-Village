# Market Village — 조립(Playable Assembly) PRD

> **목적:** 검증된 엔진 모듈들(사이클 8~11)과 기존 맵 세션을 **플레이 가능한 단일 경험**으로 조립한다.
> FINAL PRD(`Market_Village_PRD_FINAL.md`)가 *무엇을·왜*, 이 문서가 *어떻게·어떤 순서로*. 티켓은 PLANS.md 보드.

## 1. 사람 결정 (2026-07-01, awake)

- **D-E 엔진 통합 → 신 엔진 정본 + 맵=표현.** `run_loop`/`crisis_day`/`game_run`이 게임 진행을 **결정**하고, 맵·NPC·게시판은 §12.0대로 그 결정을 **연기**한다(`엔진 → 표현` 단방향). 구 reverie 매매 시뮬은 *마을 배경 NPC 연출*로 존치(폐기 아님, 수술하듯). 근거: PRD §12.0(관찰자 카메라)·§8.6.3(결정/표현 분리)·§16.0.
- **D-F UI → static_demo 확장.** 기존 Phaser 맵뷰어(`/map`) + HTML 위젯(`/demo`)을 확장. 새 프론트 신설 안 함.
- **D-G 범위 → 수직 슬라이스 먼저.** 인터뷰→하루 1개(뉴스·위기·개입·정산)→30일→카드→회차비교를 **한 흐름으로 e2e** 먼저 완성. 그다음 교류·전염·다자산으로 폭 확장.
- **D-H LLM 표현 → 키 받아 바로 연결.** `sim/llm.py`(이미 .env의 `OPENROUTER_API_KEY` 읽고 오프라인 폴백) 사용. 키 없는 동안은 결정론 템플릿으로 플레이 가능, 키 들어오면 표현만 LLM으로 교체(수치 결정 무관 §8.6.3). 🔒 실호출은 과금 게이트.

## 2. 아키텍처 (단방향 엔진→표현)

```
[인터뷰] → clone_spec(클론 = 거울)
   │
[GameRun 세션] ── 신 엔진 정본 (결정, 결정론, 회차 재현)
   │  하루씩 advance: 전날밤(회피) → 아침(뉴스3지선다) → 일과/교류 → 위기(개입 A/B/C) → 정산
   │  내부: news · trap_pipeline · resistance · crisis_day · 회계 · clone_stats · runs
   │
   ├─(state)→ [표현 계층] 맵 카메라가 클론 추적 · 연출명령(move/dialog/trade/emote) · LLM 대사
   │           static_demo: 인터뷰화면 · 상태패널 · 뉴스위젯 · 메신저 · 정산 · 회차비교 · 카드
   │
[30일 완주] → 결과카드 → [새 회차] (래포·확증편향 리셋, 카드 누적) → 회차비교(거울)
```

- **핵심 전환:** 기존 `run_loop`은 30일을 batch로 돈다. 플레이는 **하루씩 진행(step-able)** 해야 하므로 `sim/game_run.py`(상태 보유 + advance_day)로 감싼다. 구 `engine.GameSession`(맵 NPC 라운드)은 건드리지 않고 배경 연출로만.

## 3. 플레이 흐름 — 수직 슬라이스 (D-G)

먼저 "한 사람이 처음부터 끝까지 한 판을 플레이"하는 최소 경로를 e2e로:

1. 인터뷰(결정론, 키 있으면 LLM) → 클론 생성
2. GameRun 시작(클론 + 포트폴리오 + Day1)
3. 매일: 전날밤 일과+회피 → 아침 뉴스 3지선다 → (일과 연출) → 위기 트리거 시 개입 A/B/C → 밤 정산(휘둘림·스탯·스냅샷)
4. Day30 → 결과카드(2축 등급)
5. 새 회차 → 같은 운명선 다시 → 회차비교 화면(거울)

그다음 **폭 확장:** 1:1 권유·FGI 단톡방·메신저 통합(§9), 양방향 전염을 마을에(§9.3), 다자산(M/G2 §8.3), 고유이벤트 주입(§5.4), 인터뷰 LLM 대화형(§5.3).

## 5. 비기능 / 리스크

- **회귀 0:** 신 조립은 구 `engine.GameSession`·맵뷰어·SNS 시스템을 안 깨뜨린다(병렬 추가, 핸들러 직접호출 테스트 RETRO 규칙5).
- **결정론:** GameRun은 시드 고정 — 회차 재현(기둥3). LLM 표현은 결정에 영향 없음(§8.6.3).
- **하드게이트:** LLM 실호출(과금)·Mongo 영속화는 사람/키 대기. 자율 빌드는 오프라인 pytest로만 검증.
- **리스크:** step-able 리팩토링이 run_loop batch 로직과 중복되지 않게 — run_loop의 per-day 코어를 GameRun이 재사용(DRY).
