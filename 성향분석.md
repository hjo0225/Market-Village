# 성향분석 개편: MBTI 스타일 투자성향 테스트

> 목적: 현재 온보딩 7문항 진단(`backend/sim/disposition.py`, `frontend/app/emo/page.tsx`)을
> "MBTI 테스트 느낌"으로 개편한다. 단, 문항·분류의 근거는 한국 금융권이 실제로 시행하는
> **투자성향분석(투자자정보 확인서)** 에 둔다. 이 문서가 구현 스펙의 정본이다.

---

## 1. 근거 (실제 한국 투자성향분석)

### 1.1 법적·제도적 근거
- **금융소비자보호법 제17조(적합성원칙)**: 금융상품 권유 전 면담·질문으로
  **거래목적, 재산상황, 투자경험, 상품 이해도(금융지식), 위험에 대한 태도, 연령**을
  파악·확인해야 하며, 부적합 상품 권유 금지.
  - https://www.law.go.kr/LSW//lsSideInfoP.do?lsiSeq=260405&joNo=0017
- **금융투자협회 「표준투자권유준칙」** 별지 제1호 "일반투자자 투자자정보 확인서"가 실무 표준.
  - https://law.kofia.or.kr/service/law/lawFullScreenContent.do?seq=149&historySeq=1556
- **금융위 「투자성 상품 위험등급 산정 가이드라인」**(2023): 상품 위험 1~6등급(1=초고위험).
  - https://www.fsc.go.kr/po010101/79327

### 1.2 표준 문항 카테고리 (투자자정보 확인서)
실제 확인서는 다음 카테고리를 묻는다 (HMC/신한/토스증권 등 공통):

| # | 카테고리 | 예시 |
|---|---|---|
| 1 | 연령대 | 40세 이하 ~ 65세 이상 |
| 2 | 투자 가능 기간 | 6개월 이내 ~ 3년 이상 |
| 3 | 투자경험 상품 | 예·적금 ~ 선물옵션/신용거래 (5단계) |
| 4 | 금융지식 수준 | 매우 낮음 ~ 매우 높음 |
| 5 | 금융자산 대비 투자비중 | 10% 이내 ~ 40% 초과 |
| 6 | 수입원 안정성 | 안정적 유지 ~ 연금이 주 수입 |
| 7 | 손실감내도 | 원금 보전 필수 ~ 고수익 위해 고위험 감수 |

### 1.3 표준 5단계 분류와 판정 방식
문항별 차등 배점 → **합산 → 구간 판정** (예: 신한투자증권 16↓/17–24/25–32/33–40/41↑):

| 성향 | 정의 | 권유 가능 상품 위험등급 |
|---|---|---|
| 안정형 | 원금손실 불원, 예금 수준 수익 | 5등급 이하 (예·적금, MMF, 국채) |
| 안정추구형 | 손실 최소화, 이자·배당 중심 | 4등급 이하 (채권형펀드 등) |
| 위험중립형 | 위험-수익 비례 인식, 예금+α | 3등급 이하 (혼합형펀드 등) |
| 적극투자형 | 원금보전보다 수익 우선 | 2등급 이하 (주식, 주식형펀드) |
| 공격투자형 | 시장수익률 초과 추구 | 1등급 이하 (파생, 신용거래 포함 전체) |

## 2. 구현 방향성: "겉은 MBTI, 속은 투자자정보 확인서"

핵심 설계: **채점 골격(합산→5단계)은 표준 방식 그대로 유지**하고,
그 위에 MBTI식 4축 문자 코드를 **표현 레이어**로 얹는다.
기존 `declared_type`(5유형) 파이프라인 — 편향 기대값, 감정 초기화, 엔딩 리포트 —
전부 무변경으로 재사용한다.

### 2.1 MBTI 4축 (각 축은 확인서 카테고리에 대응)

| 축 | 문자 | 근거 카테고리 |
|---|---|---|
| 위험 태도 | **A**(공격 Aggressive) / **S**(안전 Safe) | 손실감내도, 투자목적 |
| 시계(호흡) | **L**(장기 Long) / **Q**(단기 Quick) | 투자기간, 매도 시점 |
| 판단 근거 | **D**(데이터 Data) / **F**(감 Feeling·군중) | 금융지식, 정보 의존처 |
| 자금 여력 | **R**(여유 Reserve) / **E**(올인 Edge) | 투자비중, 수입원, 감당능력 |

→ 2⁴ = **16타입**, 각 타입에 별명 부여 (예: `AQFE` = "불나방", `SLDR` = "돌다리 감별사",
`ALDR` = "냉혈 설계자" …). 별명 16개는 구현 시 확정.

### 2.2 문항 설계 (12문항, 선택지형 유지)

- 기존 7문항의 상황극 톤(룸메이트, −30% 등)은 유지하되, 표준 확인서 7개 카테고리를
  전부 커버하도록 **12문항으로 확장** (축당 3문항).
- 각 선택지는 두 가지 배점을 동시에 가진다:
  1. **risk 점수 1~4** (기존 raw 합산용 — 5단계 판정에 사용, Q'손실감내도' 가중치 2배 유지)
  2. **축 극성** (해당 축의 어느 문자 쪽인지 ±1)
- 축 판정: 축당 3문항 극성 합의 부호로 문자 결정 (동점 없음 — 3문항 홀수 설계).

### 2.3 결과 화면 (MBTI 결과 카드 느낌)

1. **타입 코드 + 별명 카드** (예: "AQFE 불나방") — 큰 카드, 캐릭터성 있는 한 줄 설명
2. **4축 게이지 바** (A 63% ←→ S 37% 식)
3. **공식 성향 배지**: 표준 5단계 유형 + "권유 가능 위험등급" 표 (1.3절) — 근거 노출
4. 잘 맞는/안 맞는 궁합 타입 한 줄 (MBTI 감성, 순수 표현용)
5. 공유용 이미지/문구

### 2.4 코드 변경 지점

| 위치 | 변경 |
|---|---|
| `backend/sim/disposition.py` | `QUESTIONS` 12문항으로 교체(risk 점수 + axis 극성 필드 추가), `mbti_type()` 함수 추가. `declared_type` 밴드는 12문항 raw 범위에 맞게 재산정(합산→구간 방식 동일). |
| `disposition.py` `_EXPECTED_BIAS`, `build_initial_emotion_v2` | 입력이 여전히 5유형/문항 점수이므로 매핑만 새 문항 인덱스에 맞춰 조정 |
| `frontend/app/emo/page.tsx` | 12문항 진행 UI를 MBTI 테스트풍으로: 1문항 1화면, 진행바, 결과 카드 |
| `frontend/components/` | 결과 카드 컴포넌트 신규 (2.3절) |
| `backend/sim/test_disposition.py` | 12문항 채점·축 판정·경계값 테스트 갱신 |

### 2.5 지키는 것 / 안 하는 것

- **지킴**: 합산→구간 판정(표준 방식), 5단계 명칭·정의·위험등급 매핑(표준 그대로),
  기존 2층 편향 5축·리포트 파이프라인(무변경), 손실감내도 가중치.
- **안 함**: 실명 확인서 문항 그대로 복제(게임 톤 유지), 연령/소득 등 개인정보성 직접 질문
  (상황극으로 치환), 16타입별 별도 게임 로직 분기(표현 레이어일 뿐).

---

## 3. 요약

MBTI는 포장이고 뼈대는 금소법 제17조·표준투자권유준칙의 투자자정보 확인서다.
문항 카테고리(7종)와 합산→5단계 판정을 그대로 계승해 "근거 있는 테스트"임을 결과
화면에서 명시하고, 4축 16타입은 그 위의 재미 레이어로만 동작한다. 기존 진단→편향
기대값→엔딩 리포트 파이프라인은 5유형 인터페이스를 통해 손대지 않고 재사용한다.
