# Market Village — 투자 감정 거울 라이프시뮬

> 내 투자 성향을 복제한 **클론**과 함께, 블라인드 처리된 **과거 실제 코인 시장**에서
> 10일을 산다(첫 화면에서 **3일 압축판** 선택 가능). 선택은 내가 하고, 본능은
> 클론이 보여준다. 잃는 것은 가상 시드고, 남는 것은 내 감정 패턴이다.

## 1. 문제 — 시장의 공백

초보~중급 투자자 손실의 주범은 정보 부족이 아니라 **감정 패턴**이다(급락 투매,
고점 추격, 익절 거부, 확증편향). 그런데 이 문제를 정면으로 다루는 제품이 없다:

| 기존 대안 | 실제로 주는 것 | 못 주는 것 |
|---|---|---|
| 모의투자 | 예측·매매 연습 | 가짜 돈엔 감정이 실리지 않음 — 패턴이 드러나지 않는다 |
| 투자 교육 콘텐츠 | 지식 | 아는 것과 위기 순간의 행동은 별개 |
| 트레이딩 저널 | 기록 | 스스로 쓰는 기록은 합리화로 오염된다 |
| 실전 | 진짜 학습 | 수업료가 실제 돈, 그마저 "시장 탓"으로 휘발 |

**공백: 내 감정 패턴을 — 내 돈을 걸지 않고 — 제3자의 눈으로 — 반복 관찰·교정하는 장치.**

## 2. 가치 제안

**"내 성향을 복제한 클론의 본능과 열흘을 싸우고, 감정 패턴만 가져간다."**

1. **거울** — 설문으로 복제된 클론과 하루하루를 산다. 매 결정점(단톡방 반응)에
   **「클론의 본능」 배지**가 내 성향이 미는 선택지를 먼저 보여주고, 최종 선택은
   내가 한다. 본능을 거스르면 불안 비용을 치르고 그 규율이 집계되며(규율
   리포트), 매매 규모는 클론의 감정 상태에 비례한다 — 충동이 클수록 크게
   지른다. 엔딩 리포트가 **선언한 성향 vs 실제 고른 행동**의 간극을 보여주는
   순간, 처음으로 패턴이 보인다.
2. **수업료 0원의 실전 감정** — 시장은 **실명 코인(BTC·XRP·DOGE·USDT)의
   과거 실제 데이터**(코인마다 서로 다른 실제 시기). 단, "언제였는지"만
   블라인드(운명선)라 힌드사이트가 차단된다. 가격은 이미 정해져 있어 예측
   게임으로 변질될 수 없고, 선택과 감정만이 결과를 가른다. 매일 밤 정산
   캐스케이드가 "선택 → 시세 변동 → 내 자산 → 감정"을 단계로 보여주고,
   **원인 카드**가 "어제 그 선택이 없었다면 오늘 −2.4% 대신 −0.8%"처럼
   오늘 성과를 어제의 선택과 반사실 비교로 연결해 준다. 엔딩에서야 시기가
   공개된다 — "당신의 열흘은 사실 2021년 4월이었다."
3. **같은 시험지 재시험** — 운명선이 회차 무관 고정이라, 다시 시작하면 **같은
   10일을 다시 산다**. "저 결과를 뒤집고 싶다"는 후회가 리텐션 엔진이고,
   뒤집는 순간이 학습의 증거다. 지금은 결과 카드 2×2 등급이 회차 비교 축이고,
   회차 기록 겹쳐보기 UI는 로드맵.
4. **평가가 가치관을 강제한다** — 결과는 수익률이 아니라 수익률×감정통제
   2축: 🏆 고수 / 💸 벼락부자형(운빨) / 🛡 강철멘탈형(과정 승리) / 🔥 호구.
   회차 내내 지향할 목표는 **통제 티어 5단계**(새싹→불개미 졸업→평정 수련→
   강철 멘탈→마을의 현자) — 수익률이 아니라 감정 통제로만 오른다. "수익
   났는데 왜 최고 등급이 아니지?"라는 순간이 이 제품의 교육이 작동하는
   순간이다.
5. **육성 게임의 하루** — 프린세스메이커식 **일과 편성**: 행동력 안에서
   오전/오후/저녁 장소를 배치하고, 배치 전에 예상 감정 변화와 그 장소에
   나타날 NPC 얼굴을 미리 본다. 점심 단톡방(게시판) 이벤트는 6개 선택지
   풀에서 매일 3개가 변주 등장. 설문 전후 스토리 씬과 진단 근거 카드
   ("Q5에서 '더 산다'를 골라 위험감수 +2")가 세계관과 분류 기준을 서사로
   전달한다.

## 3. BM 가설 (검증 전 — 완주율·재회차율 데이터로 검증한다)

비용 구조가 BM을 받친다: 게임 수치·대사는 전부 **오프라인 결정론 엔진**(정적
텍스트)이라 무료 유저의 한계비용이 ~0이다. LLM은 표현 계층에만 opt-in — 현재
라이브에서는 **엔딩 리포트 서술 1곳**(게임당 1회 생성 후 박제, 서버 게이트
`MV_ALLOW_LLM`·일일 한도·프롬프트 캐시·결정론 폴백 내장)뿐이다. 즉 **무료
티어가 원가를 거의 만들지 않고, 유료 가치는 LLM 개인화에 집중**시킬 수 있다.
**3일 압축판(약 10분)이 무료 최상단 퍼널** — 첫 체험 완주를 쉽게 만들어 10일
정식판(→ 유료 후보)으로 잇는다.

| 축 | 내용 | 근거 · 현황 |
|---|---|---|
| **B2C 프리미엄** | 무료: 정적 대사·기본 리포트로 풀 게임(=현재 라이브 전체). 유료(로드맵): LLM 실시간 개인화(심층 인터뷰 복제 고도화·클론 독백·1:1 대화 다양화) + 정밀 감정 리포트 | 원가가 LLM 콜에 국한(일 한도·캐시로 상한 고정 — 게이트 구현 완료) |
| **콘텐츠 시즌** | 새 운명선 팩(새 블라인드 구간 = 새 시험지), 시그니처 드라마(밈 광풍·대폭락·페그 붕괴) 회차 | 운명선 1개 = 데이터 박제 1회, 제작 한계비용 낮음(현재 1팩 라이브) |
| **B2B 제휴** | 거래소·증권사의 투자자 보호/온보딩 교육 모듈로 화이트라벨 | "예측·투자권유가 아니라 감정 교육"이라는 포지션이 규제 친화적 — 금융사가 붙기 쉬운 드문 게임 |
| **컬렉션·소셜** | 결과 카드 수집(회차 연대기), 카드 공유 (로드맵 — 현재는 진단 카드 공유 복사만) | 획득 비용 0, 재회차 동기 강화 |

**Non-BM(하지 않는 것):** 투자 자문·시그널 판매 ✗ (제품 철학과 규제 리스크
정면충돌) · 예측 리더보드 ✗ (예측 게임화는 설계 기둥 위반) · 개인 감정 데이터
판매 ✗.

## 4. 왜 이 구조가 해자인가

- **표현/엔진 분리(단방향)** — 수치는 결정론 엔진, LLM은 연기만. 무료 티어
  원가 ~0과 "같은 시험지 재현"(회차 비교)이 전부 이 분리에서 나온다. 베끼려면
  LLM 중심 설계를 버려야 해서 따라 하기 어렵다.
- **복제 품질이 곧 제품력** — 현재는 7문항 성향 진단이 클론의 본능(배지·규율
  판정)을 만들고, 우회 인터뷰(시나리오·과거 경험 발굴)로 취약점·합리화 언어까지
  뽑는 심층 복제가 로드맵(프리미엄 후보). 복제 깊이의 차이가 체감 품질 차이.
- **의도적 설계 시장** — 현실 시뮬 경쟁(무한 군비경쟁)에 들어가지 않는다.
  판단 기준은 "감정 인식에 기여하는가" 하나.

## 5. 현재 상태 (2026-07-11)

- **Live** — AWS EC2 + Docker Compose 4컨테이너, main 머지 → 테스트 게이트 →
  자동 배포(GitHub Actions). 접속 주소는 `HANDOFF.md` 전달 환경 참고.
- **완성 범위** — 타이틀→**모드 선택(3일 압축판 / 10일 정식판)**→프롤로그→
  성향 설문 7문항→진단 근거 카드→실명 코인 배분→**일일 루프**(아침 내레이션·
  일과 편성·단톡방 6지선다 변주·NPC 체인·장소 딜레마·정산 캐스케이드+원인
  카드·통제 티어)→엔딩 5종+시기 공개+선언 vs 실제 행동 편향 리포트.
  백엔드 전 테스트(510+) 오프라인·결정론 green.
- **3일 압축판** (심사·첫 체험용, 약 10분) — 운명선을 압축 드라마 구간으로
  당겨 **급등(유혹)→급락(시련)→반등(검증)** 사흘 아크를 산다. NPC 체인 발동
  확률 상향(0.95)으로 짧은 회차에도 관계 이벤트 보장, 히든 엔딩(E5)은 10일
  정식판 전용 보상으로 유지(압축 회차는 2×2 엔딩으로 갈린다). "열흘/사흘"
  표기는 전 구간 일수 대응.
- **다음 검증** — 10일 밸런스(행동력·티어 곡선), 3일→10일 전환율·완주율/
  재회차율 측정, 1:1 대화 LLM 다양화(프리미엄 후보 기능).
- **타겟/발표 자료** — `scenariodoc/`(로컬 전용, gitignore): 타겟_피처매칭
  (타겟 정의·피처 매칭·사용자 플로우), 전체_시나리오_일상물, 구현 스펙 v1~v3.

## 6. 개발자용 요약

```powershell
# backend (PowerShell/cmd — Git Bash는 WinError 10014 주의)
cd backend && python -m pip install -r requirements.txt
python -m uvicorn market_live_server:app --port 8100

# frontend
cd frontend && npm ci && npm run dev   # http://localhost:3000 → /emo
```

- MongoDB·LLM 키 불필요(전부 폴백). LLM은 `.env`에 `OPENROUTER_API_KEY`,
  서버 게이트 `MV_ALLOW_LLM`·일일 한도 `MV_LLM_DAILY_MAX`.
- 테스트: `cd backend && python -m pytest -q` (오프라인·결정론) ·
  `cd frontend && npx tsc --noEmit`.
- CI/CD: PR → 테스트만(ci.yml) / main 머지 → 테스트 통과 시 EC2 자동 배포
  (deploy.yml, ~5-10분). 맵 에셋 `environment/`는 gitignore — 서버 수동 업로드.
- 아키텍처: `EmoGameRun`(라이브 게임 경로, `backend/sim/emo_game.py`) →
  표현 계층(Next.js `/emo` + `/map` iframe 배경). 상세 설계는 `Market_Village_PRD_FINAL.md`
  (마스터 PRD), 작업 상태는 `PLANS.md` · `CHECKLIST.md` · `RETRO.md` ·
  `HANDOFF.md`.
