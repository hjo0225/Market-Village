# Market Village — 투자 감정 거울 라이프시뮬

> 내 투자 성격을 복제한 AI **클론**이 블라인드 처리된 **과거 실제 코인 시장**에서
> 10일을 산다. 나는 클론을 조종할 수 없다 — 지켜보고, 흔들 수만 있다.
> 잃는 것은 클론이고, 배우는 것은 나다.

## 1. 문제 — 시장의 공백

초보~중급 투자자 손실의 주범은 정보 부족이 아니라 **감정 패턴**이다(급락 투매,
고점 추격, 익절 거부, 확증편향). 그런데 이 문제를 정면으로 다루는 제품이 없다:

| 기존 대안 | 실제로 주는 것 | 못 주는 것 |
|---|---|---|
| 모의투자 | 예측·매매 연습 | 가짜 돈엔 감정이 실리지 않음 — 패턴이 드러나지 않는다 |
| 투자 교육 콘텐츠 | 지식 | 아는 것과 위기 순간의 행동은 별개 |
| 트레이딩 저널 | 기록 | 스스로 쓰는 기록은 합리화로 오염된다 |
| 실전 | 진짜 학습 | 수업료가 실제 돈, 그마저 "시장 탓"으로 휘발 |

**공백: 내 감정 패턴을 — 내 돈을 걸지 않고 — 제3자의 눈으로 — 반복 관찰·교정하는 장치.**

## 2. 가치 제안

**"당신의 투자 습관을 복제한 클론이 대신 잃어주고, 당신은 패턴만 가져간다."**

1. **거울** — 인터뷰로 복제된 클론이 내 약점 그대로 매매한다. 조종은 불가능하고
   환경(뉴스 선택·만남·위기 개입)만 바꿀 수 있다. 내 못난 매매를 남 일처럼
   보게 되는 순간, 처음으로 패턴이 보인다.
2. **수업료 0원의 실전 감정** — 시장은 블라인드 처리된 **과거 실제 데이터**
   (운명선). 가격은 이미 정해져 있어 예측 게임으로 변질될 수 없고, 클론의
   감정만이 결과를 가른다. 매일 밤 "「패닉 셀」에 휩쓸려 전량 매도 → 시세
   -3.2% → 자산 -1.1%"로 감정→행동→결과가 문장으로 복기된다.
3. **같은 시험지 재시험** — 같은 10일을 다시 살며(NG+) 1회차의 나와 2회차의
   나를 겹쳐 본다. "저 결과를 뒤집고 싶다"는 후회가 리텐션 엔진이고, 뒤집는
   순간이 학습의 증거다.
4. **평가가 가치관을 강제한다** — 결과는 수익률이 아니라 수익률×감정통제
   2축: 🏆 고수 / 💸 벼락부자형(운빨) / 🛡 강철멘탈형(과정 승리) / 🔥 호구.
   "수익 났는데 왜 최고 등급이 아니지?"라는 순간이 이 제품의 교육이 작동하는
   순간이다.

## 3. BM 가설 (검증 전 — 완주율·재회차율 데이터로 검증한다)

비용 구조가 BM을 받친다: 게임 수치는 전부 **오프라인 결정론 엔진**이라 무료
유저의 한계비용이 ~0이고, LLM은 표현(대사)에만 opt-in으로 쓴다(콜 한도·캐시
내장). 즉 **무료 티어가 원가를 거의 만들지 않고, 유료 가치는 LLM 개인화에
집중**시킬 수 있다.

| 축 | 내용 | 근거 |
|---|---|---|
| **B2C 프리미엄** | 무료: 오프라인 대사·기본 리포트로 풀 게임. 유료: LLM 실시간 개인화(심층 인터뷰 고도화·클론 독백·1:1 대화 다양화) + 정밀 감정 리포트 | 원가가 LLM 콜에 국한(일 한도·캐시로 상한 고정) |
| **콘텐츠 시즌** | 새 운명선 팩(새 블라인드 구간 = 새 시험지), 시그니처 드라마(밈 광풍·대폭락·페그 붕괴) 회차 | 운명선 1개 = 데이터 박제 1회, 제작 한계비용 낮음 |
| **B2B 제휴** | 거래소·증권사의 투자자 보호/온보딩 교육 모듈로 화이트라벨 | "예측·투자권유가 아니라 감정 교육"이라는 포지션이 규제 친화적 — 금융사가 붙기 쉬운 드문 게임 |
| **컬렉션·소셜** | 결과 카드 수집(회차 연대기), 카드 공유 | 획득 비용 0, 재회차 동기 강화 |

**Non-BM(하지 않는 것):** 투자 자문·시그널 판매 ✗ (제품 철학과 규제 리스크
정면충돌) · 예측 리더보드 ✗ (예측 게임화는 설계 기둥 위반) · 개인 감정 데이터
판매 ✗.

## 4. 왜 이 구조가 해자인가

- **표현/엔진 분리(단방향)** — 수치는 결정론 엔진, LLM은 연기만. 무료 티어
  원가 ~0과 "같은 시험지 재현"(회차 비교)이 전부 이 분리에서 나온다. 베끼려면
  LLM 중심 설계를 버려야 해서 따라 하기 어렵다.
- **복제 품질이 곧 제품력** — 우회 인터뷰(시나리오·과거 경험 발굴)로 6함정
  취약점·합리화 언어까지 뽑는 클론 스펙이 재미의 뿌리. 얕은 설문 복제와의
  차이가 체감 품질 차이.
- **의도적 설계 시장** — 현실 시뮬 경쟁(무한 군비경쟁)에 들어가지 않는다.
  판단 기준은 "감정 인식에 기여하는가" 하나.

  자동 배포(GitHub Actions). 접속 주소는 `HANDOFF.md` 전달 환경 참고.
- **완성 범위** — 인터뷰→클론 생성→10일 루프(뉴스 3지선다·마을 만남·게시판·
  위기 개입·매매 서사 정산)→결과 카드→회차 비교. 백엔드 전 테스트
  오프라인·결정론 green.
- **다음 검증** — 10일 밸런스(이벤트 빈도), 완주율/재회차율 측정, 1:1 대화
  LLM 다양화(프리미엄 후보 기능).

## 6. 개발자용 요약

```powershell
# backend (PowerShell/cmd — Git Bash는 WinError 10014 주의)
cd backend && python -m pip install -r requirements.txt
python -m uvicorn market_live_server:app --port 8100

# frontend
cd frontend && npm ci && npm run dev   # http://localhost:3000/play
```

- MongoDB·LLM 키 불필요(전부 폴백). LLM은 `.env`에 `OPENROUTER_API_KEY`,
  서버 게이트 `MV_ALLOW_LLM`·일일 한도 `MV_LLM_DAILY_MAX`.
- 테스트: `cd backend && python -m pytest -q` (오프라인·결정론) ·
  `cd frontend && npx tsc --noEmit`.
- CI/CD: PR → 테스트만(ci.yml) / main 머지 → 테스트 통과 시 EC2 자동 배포
  (deploy.yml, ~5-10분). 맵 에셋 `environment/`는 gitignore — 서버 수동 업로드.
- 아키텍처: `GameRun`(유일한 게임 경로, `backend/sim/`) → 표현 계층(Next.js
  `/play` + Phaser `/map` 배경). 상세 설계는 `Market_Village_PRD_FINAL.md`
  (마스터 PRD), 작업 상태는 `PLANS.md` · `CHECKLIST.md` · `RETRO.md` ·
  `HANDOFF.md`.
