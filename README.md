# Market Village — 투자 감정 거울 라이프시뮬

> 내 투자 성격을 복제한 AI **클론**이 블라인드 처리된 **과거 실제 코인 시장**에서
> 10일을 산다. 플레이어는 클론을 직접 조종하지 못한다 — **뉴스 3지선다 · 일과
> 회피 · 대화 권유 · 위기 개입**으로 감정만 흔들 수 있다.
>
> **핵심 비유: 플레이어는 조종자가 아니라, 자기 복제를 관찰·자극하는 실험자다.**

## 문제 — 왜 만들었나

초보~중급 투자자의 손실 주범은 정보 부족이 아니라 **감정 패턴**이다 — 급락에
던지는 공포, 고점에 올라타는 FOMO, 익절 못 하는 탐욕, 듣고 싶은 뉴스만 믿는
확증편향. 이 패턴은 ①자기 눈에 안 보이고 ②실전에서 배우면 수업료가 실제
돈이며 ③지식이 아니라 행동의 문제라 차트 공부로는 안 고쳐진다.

**목표 (Primary):** 자기 투자 감정 패턴을 인식하고, 위기에서 더 침착하게
행동하도록 학습. **(Secondary)** 그 학습을 라이프시뮬·육성 게임의 재미로 전달.
**(Non-Goals)** 수익률 극대화 ✗ · 차트/지표 교육 ✗ · 클론 능력치 육성 ✗ —
키우는 건 클론이 아니라 **플레이어의 자기 인식**이다.

## 해결 — 설계 기둥 6 (기능 추가·변경 시 깨면 안 됨)

1. **거울** — 클론은 "거울에 비친 나". 직접 조종하면 거울이 깨진다. 플레이어는
   클론을 _둘러싼 환경_에만 개입한다.
2. **거울은 회차 비교에서 나온다** — 한 판 안에선 능동 개입 OK. "1회차의 나 vs
   2회차의 나"를 겹쳐 볼 때 거울이 발생한다.
3. **실험 구조** — 통제변인(가격 곡선·이벤트·클론 초기 성격)은 고정,
   조작변인(뉴스·회피·권유·개입)만 플레이어가 바꿔 결과를 관찰한다.
4. **과정 > 결과** — 1순위 지표는 수익률이 아니라 "감정에 휘둘렸는가". "잘
   판단했는데 손해 보는 회차"를 의도적으로 포함한다.
5. **의도적 설계 시장** — 현실 시뮬이 아니다. 판단 기준은 현실성이 아니라
   "감정 인식에 기여하는가".
6. **표현/엔진 분리, 단방향** — 모든 결정은 엔진(순수·결정론)에서. 맵·캐릭터·
   LLM 대사는 연기만 한다(`엔진 → 표현`). LLM이 수치를 역으로 바꾸지 않는다.

## 게임 구조 (3계층)

- **메타:** 회차 반복(NG+) — 같은 시험지(고정 운명선)를 다시 살고, 회차 간
  비교가 곧 학습.
- **세션:** 한 회차 = 고정 10일.
- **일일:** 하루 ≈ 5분 — 전날밤(회피) → 아침(뉴스 3지선다) → 낮(일과·만남·
  게시판) → 위기(개입 A/B/C → 저항 판정) → 밤(회계·매매 서사 정산).

## 핵심 시스템 — 무엇이 어떻게 구현돼 있나

| 시스템 | 설계 의도 → 구현 |
|---|---|
| **클론 생성 (§5)** | 모든 재미의 뿌리. 대화형 인터뷰(직접 안 묻고 시나리오·과거 경험으로 우회, LLM opt-in) → 6함정 취약점·뉴스톤 계수·합리화 언어·고유 이벤트를 가진 클론 스펙. 채점 공식은 비공개(조작 방지). + 시작 포트폴리오 4유형 배분·에이전트 이름 짓기. |
| **시장 (§6)** | 실제 업비트 일봉을 종목·시기 블라인드로 박제한 **운명선**(첫날 종가=100 정규화, 회차 불변). 세상 가격은 고정, 클론 성적만 행동 따라 바뀐다 — 회차 비교가 성립하는 근거. 클론은 가격 수용자: 손익은 "가격을 움직여서"가 아니라 "잘못된 순간에 매매해서" 난다. |
| **뉴스 3지선다 (§7)** | 뉴스는 가격이 아니라 클론의 **심리**를 흔드는 레버(가격 불변). 명백한 정답 없음 — 안전하게 둘 것이냐, 취약점을 시험할 것이냐의 딜레마. 하루 1회 선택 잠금. |
| **6함정+확증편향 (§8)** | F1 급락패닉·F2 멘탈마모(정답=버티기) vs G1 익절거부(정답=팔기) — 정반대라 암기가 아니라 상황 판단력을 요구. G2 몰빵·M1 추격매수·M2 막차불안 + 확증편향 잠금(손실 포지션에 가두는 후회 재생산). 매도 대금의 행선지로 함정을 읽는다. |
| **교류 (§9)** | 클론을 빚는 메인 레버. 트레이더 NPC 8종이 같은 시장을 살고, 일과 동선이 겹치면 자연 발생 만남(배치 ✗, 옆 칸까지 와서 마주보고 대화). 폰 = 통합 무대(게시판·이벤트 타임라인·메신저). 게시판 여론은 "오른다/내린다 힌트"가 아니라 **군중 쏠림 경고**로 프레임 — 쏠림 추종이 곧 함정이므로. |
| **저항 (§11)** | 황금률: 저항 **원인은 100% 투명**(무력감 방지), 극복은 **결코 100% 불가**(꼭두각시 방지). 기질×상태×개입 3층의 곱. 위기 개입과 1:1 권유가 **같은 래포 풀**을 공유 — 남발하면 정작 중요할 때 안 먹힌다. 실패 시 이유 표시로 학습 유도. |
| **감정 스탯 (§10)** | 육성 게임 스탯 UI를 쓰되 "올리는" 게 아니라 "무너지는 걸 견디는" 게임. 6함정 개별 저항 + 멘탈회복 7스탯(PRD 4스탯에서 구현 중 정밀화). 낮은 스탯 = 높은 함정 발동 확률. |
| **매매 서사** | 복기 장치. 매일 밤 "「패닉 셀」의 감정에 휩쓸려 전량 매도했어요 → 시세 -3.2% → 내 자산 -1.1%"를 정산 모달로, 종목별 시세·비중과 마을의 매매 동태까지 한눈에. 발자취·이벤트 타임라인에 일별 누적. |
| **결과 카드 (§14)** | 수익률 × 감정통제 **2축 평가**: 🏆 고수 / 💸 벼락부자형(운빨) / 🛡 강철멘탈형(과정 승리) / 🔥 호구. "수익 높은데 왜 최고가 아니지?"가 기둥 4(과정>결과)의 체감 장치. |
| **회차 비교 (§13.6)** | 거울의 핵심 산출물 — 이전 회차와 같은 날짜로 겹쳐, 행동이 갈린 날만 짚어 보여준다. |

**플레이어 정보 범위(거울용, 적게 보여줌):** 미래 힌트 절대 없음 · 클론
속마음은 결정적 순간만 개방 · 개입 성공 확률은 사전 비공개(사후 이유만).

## 빠른 시작 (로컬)

MongoDB·LLM 키 **불필요**(전부 폴백/자기완결). 터미널은 **PowerShell 또는
cmd** — Git Bash에서 uvicorn을 직접 돌리면 Windows 소켓 문제(`WinError
10014`)로 안 뜰 수 있다.

```powershell
cd backend
python -m pip install -r requirements.txt
python -m uvicorn market_live_server:app --port 8100
```

```powershell
cd frontend
npm ci
npm run dev   # http://localhost:3000/play — 유일한 게임 진입점
```

- **`/play`** — 인터뷰 → 하루 루프 → 10일 완주 → 결과 카드 → 회차 비교까지 한 화면.
- `http://localhost:8100/map` — Phaser 마을 맵 뷰어(배경 연출 전용 — 기둥 6, 게임 진행은 결정 안 함).

### LLM 표현 계층 (선택)

대사 표현에 LLM을 쓰려면 레포 루트 `.env`에 `OPENROUTER_API_KEY=sk-or-...`
(또는 OS 환경변수 `OPENAI_API_KEY`). 없으면 오프라인 결정론 템플릿으로 동작.
서버 게이트 `MV_ALLOW_LLM`·일일 한도 `MV_LLM_DAILY_MAX`로 과금 방어.

## 배포 · CI/CD (2026-07-06 라이브)

**main에 머지하면 서버에 자동 반영된다.**

```
PR 열기 → ci.yml (pytest + tsc, 테스트만)
main 머지 → deploy.yml: 테스트 게이트 → EC2에 SSH → docker compose 재빌드 (~5-10분)
```

- 인프라: AWS EC2 1대(서울) + Docker Compose 4컨테이너 — mongo(인증·외부
  미노출) / backend / frontend / caddy(:80). 접속 주소·인스턴스 정보는
  `HANDOFF.md` 전달 환경 참고.
- 테스트가 깨지면 배포가 자동으로 막힌다. 수동 재배포는 Actions 탭 → Deploy → Run workflow.
- 맵 에셋 `environment/`는 gitignore — 서버에 수동 업로드(변경 시 재복사 필요).

## 아키텍처

```
[인터뷰] → clone_spec(클론 = 거울)
   │
[GameRun] ── 유일한 게임 경로(sim/game_run.py) — 하루씩 진행, 결정론, 회차 재현
   │  전날밤(회피) → 아침(뉴스 3지선다) → 위기(개입 A/B/C·저항판정) → 정산
   │  내부: news · trap_pipeline · resistance · crisis_day · 회계 · clone_stats
   │
   └─(state)→ 표현 계층(sim/presentation.py) — 연출명령 + 대사(LLM opt-in)
              Next.js `/play` UI + Phaser `/map` 배경(좌표 브릿지 §12.0)
```

`/control/game/*`가 게임 진행 경로, `/map`은 그 state를 보여주는 배경 연출
경로다(단방향 — 기둥 6). Generative Agents는 맵·이동·행동 **표현만** 차용,
인지 엔진은 차용하지 않았다. 상세 설계는 `Market_Village_PRD_FINAL.md`,
조립 흐름은 `ASSEMBLY_PRD.md`.

## 시뮬레이션 모듈 (`backend/sim/`)

| 모듈 | 기능 |
|---|---|
| `game_run.py` | 하루씩 진행 가능한 회차 세션(GameRun) — 플레이의 근간 |
| `run_loop.py` | 전체 회차 batch 실행(회차 비교용) + `step_day` 공유 코어 |
| `crisis_day.py` | 하루치 결정 파이프라인 조립(뉴스→컨텍스트→함정→회계+스탯) |
| `traps.py` | 6함정(F1/F2/G1/G2/M1/M2) 분류 + 트리거 감지 |
| `trap_pipeline.py` | 함정 판정 STEP1~6 + 확증편향 잠금 + 격화 양날 |
| `resistance.py` | 개입 전략 A/B/C + 래포 + 4요소 저항식 |
| `news.py` | 뉴스 3지선다(가격 불변, 클론 심리만) |
| `clone_spec.py` | 인터뷰 → 6함정 취약점·뉴스톤 계수·합리화 언어·고유 이벤트 |
| `fate_line.py` | 고정 운명선(블라인드 과거 데이터) + 가격 하이브리드(90%고정+10%출렁임) |
| `trade.py` | 가격수용자 회계(평단·실현손익·자금행선지) |
| `clone_stats.py` | 클론 7스탯(6함정 개별 저항 + 멘탈회복) |
| `runs.py` | 회차(NG+) 3계층 저장 + 회차 비교(§13.6 거울) |
| `result_card.py` | 결과 카드 2축 평가(고수/벼락부자형/강철멘탈형/호구) |
| `day_loop.py` | 8슬롯 하루·일과 셔플·확률 외란 |
| `avoidance.py` | 전날밤 회피 + 클론의 NPC 선택(약점 끌림 — 길 C) |
| `personas.py` | 트레이더 NPC 8종(자극 4 + 도움 4) + 마을 프리셋 |
| `social.py` | 설득/FGI 개입 + 게시판 문구 풀 |
| `board_v2.py` | 이벤트 게시판 대화 생성(오프라인 결정론 + LLM opt-in) |
| `meeting_talk.py` | 1:1 만남 대사(결정론, 폰 채팅 연출용) |
| `agent_state.py` | NPC 감정 누적·개별 래포·기억 발화 |
| `sentiment.py` | 감정 기여도 계산 |
| `presentation.py` | 엔진 결정 → 연출명령+대사(LLM opt-in) |
| `interview.py` / `interview_llm.py` | 인터뷰 질문·클론 스펙 생성 |
| `db.py` | 게임 세션 영속화(MongoDB, 실패시 인메모리 폴백) |
| `tuning.py` | 전 수치 상수 단일 소스 |

## 테스트

```bash
cd backend
python -m pytest -q
```

전부 오프라인·결정론(LLM 키·네트워크·MongoDB 불필요). `conftest.py`가
LLM 실호출을 자동 차단한다. 프론트는 `cd frontend && npx tsc --noEmit`.

## 진행 상태 · 알려진 리스크

`PLANS.md`(작업 보드) · `CHECKLIST.md`(실행 트레이스) · `RETRO.md`(회고 규칙) ·
`HANDOFF.md`(전달 상태 — coded/merged/live) 참고.

- **live:** 수직 슬라이스(인터뷰→하루 루프→정산→카드→회차비교) + 마을 표현
  계층(맵 걷기·만남·게시판·폰 UI·매매 서사) + AWS CI/CD.
- **확장 예정:** 1:1 대화 LLM 다양화, 10일 밸런스 튜닝(이벤트 빈도가 30일
  설계 기준), 격화 양날·하드모드(P2), IAM 유저 전환·도메인+TLS.
- **핵심 리스크(PRD §17):** 복제 품질(인터뷰가 얕으면 거울 붕괴 — 최우선
  깊이) · 뉴스 무의미감(레버임을 UX로 명확히) · 회차 지겨움(빨리감기+얽힌
  연쇄) · 수치 균형(모든 값 잠정, 플레이 테스트로 보정).
